# 视觉 AI 模块

## 你的任务

为文章生成两类视觉素材的提示词：
- 封面图：3 组差异化创意
- 内文配图：3-6 张，按段落匹配

无论用户有没有生图 API，你都必须输出完整的提示词交付物。默认将结果保存到该篇文章目录下的 `image-prompts.md`。如果没有 API，这个文件本身就是最终交付物。

---

## 图位是一体化产物

- 图位不是“后面再说”的补充项，而是正文结构的一部分
- `article.md`、`image-prompts.md`、本地图片文件必须一一对应
- 封面固定文件名为 `cover-wide.jpg` 和 `cover-square.jpg`
- 内文配图固定文件名为 `img-01.jpg`、`img-02.jpg`、`img-03.jpg`……
- 每个图项都必须同时给出：
  - 槽位编号
  - 目标文件名
  - 插入位置
  - alt 文案
  - 图例说明
  - 对应提示词
- 正文必须插入对应 Markdown 图片引用，例如：

```markdown
![配图 1：这里是 alt 文案](img-01.jpg)
```

- 无论是否有生图 API，都先创建同名占位图文件
- 如果后续拿到真实图片，直接覆盖同名文件，不改 Markdown 路径
- 这三件事必须视为一个整体：
  - 正文里的图位
  - 目录里的图片文件
  - `image-prompts.md` 里的提示词和说明

## 总原则

### 1. 提示词必须是中文

- 所有正向提示词、负向提示词、版式说明、数据说明，都用中文写
- 输出的是可直接投喂给中文生图工具的完整提示词，不是几个松散关键词
- 句子要完整，信息要具体，要让另一个模型一眼知道“这张图要长什么样、写什么字、放什么数据”

### 2. 数据必须具体，不能空泛

如果图片里涉及数据、图表、榜单、对比、时间线、结论卡片，必须明确写出：
- 具体数值
- 单位
- 时间
- 对比关系
- 来源名
- 图表类型
- 每个数据项的大致排版位置

禁止写：
- “一些数据”
- “若干图表”
- “几组对比”
- “显示增长趋势”

要写成：
- “左侧柱状图显示 2024 年 58%，2025 年 73%，右上角角标写‘同比 +15pct’”
- “底部时间轴写 2023、2024、2025、2026，节点由浅蓝到深蓝渐变”

### 3. 中文必须清晰可读，绝不能乱码

只要图中出现文字，必须在提示词中明确要求：
- 仅使用简体中文
- 中文必须准确、完整、清晰可读
- 禁止乱码、错别字、异体字、繁简混用、无意义符号、拼音代替汉字
- 禁止把中文渲染成随机笔画、方框字、伪汉字

建议直接写进提示词的硬约束：

```text
画面中的所有文字仅使用简体中文，中文必须自然、准确、清晰可读，不能出现乱码、错别字、异体字、拼音替代汉字、繁简混用、无意义符号。
```

### 4. 文字不要贪多，但出现时必须“少而准”

- 封面图：优先 1 组主标题 + 1 组副标题/角标
- 信息图：优先 3-8 组关键信息标签
- 避免满屏密密麻麻小字
- 宁可少写几组字，也不要牺牲可读性

### 5. 如果工具的中文能力不稳，要同时给兜底方案

除了主提示词，再额外给出：
- 后期覆字说明
- 需要手工覆盖的中文文案清单
- 安全留白区域说明

注意：兜底方案是补充，不是替代。主方案仍然要明确要求“中文直出且不能乱码”。

### 6. 只能使用文章里已经确认过的真实信息

- 只使用 Step 3b 已确认的真实素材和 Step 5 已确认的最终稿内容
- 不得为了好看补造数字、来源、排名、机构名
- 如果正文没有足够数据，就做场景图或概念图，不强行造图表

## 一、封面图

### 双封面硬规则

- 每篇文章必须同时交付两张封面
- 横版封面：`cover-wide.jpg`
- 方形封面：`cover-square.jpg`
- 横版比例固定为 **2.35:1**
- 方形比例固定为 **1:1**
- 两张图必须共用同一视觉主题、同一主标题逻辑、同一色调体系
- 方形封面不能只是横版封面的机械裁切，必须重新适配主体位置、文字层级和留白结构
- 默认发布到公众号草稿箱时使用 `cover-wide.jpg`
- `cover-square.jpg` 作为该篇文章的配套封面物料，保存在文章目录中

### 创意策略

每次至少给 3 组创意，且必须差异明显：

**创意 A：观点冲击型**
- 用一个强视觉隐喻表达文章核心判断
- 适合观点文、热点文、趋势判断文

**创意 B：场景氛围型**
- 用一个具象场景承接文章情绪或问题意识
- 适合故事文、案例文、人物文

**创意 C：信息数据型**
- 用简洁的信息图/数字卡片/图表拼贴传递关键信息
- 适合测评、清单、盘点、复盘、行业分析

### 封面图输出格式

每组创意必须输出以下字段：

```markdown
### 封面创意 A: {创意名称}
- 横版槽位编号：cover-wide
- 横版目标文件名：cover-wide.jpg
- 横版比例：2.35:1
- 方形槽位编号：cover-square
- 方形目标文件名：cover-square.jpg
- 方形比例：1:1
- Markdown 引用：封面不插入正文，但发布时默认使用 `{article_dir}/cover-wide.jpg`
- 视觉策略：{观点冲击型 / 场景氛围型 / 信息数据型}
- 适合原因：{为什么这篇文章适合这条创意}
- 画面核心：{一句话说清这张图最重要的视觉中心}
- 必须出现的中文：
  - 主标题："{中文标题}"
  - 副标题/角标："{如果需要}"
- 必须出现的数据：
  - {数据1：数值、单位、时间、来源、位置}
  - {数据2：数值、单位、时间、来源、位置}
- 横版构图：{2.35:1 横版、主体位置、留白位置、信息区位置}
- 方形构图：{1:1 方形、主体位置、留白位置、信息区位置}
- 版式：{主标题放哪里，数据卡放哪里，图标放哪里；横版和方形分别说明}
- 色调与材质：{主色、辅色、质感、光影、风格}
- 中文排版要求：{字体气质、字重、字号层级、行数控制}
- 横版正向提示词：
  "{一整段超详细中文提示词，必须包含 2.35:1 构图、主体、材质、光线、中文文字要求、数据内容、版式约束}"
- 方形正向提示词：
  "{一整段超详细中文提示词，必须包含 1:1 构图、主体、材质、光线、中文文字要求、数据内容、版式约束}"
- 负向提示词：
  "{明确写出不要出现的内容，尤其是乱码、错别字、密集小字、低清晰度、杂乱背景}"
- 后期覆字说明：{如果直出中文风险高，说明后期该怎么覆盖}
```

### 封面图写法要求

- 横版必须指定 **2.35:1**
- 方形必须指定 **1:1**
- 如果文章是数据型内容，封面应至少带 1-3 个最关键数字
- 中文标题尽量控制在 4-12 个字内，副标题尽量控制在 6-18 个字内
- 标题和数据卡必须留出清晰可读空间
- 封面不是论文摘要，不要把全部结论塞进去
- 如果使用图表，优先柱状图、折线图、数字卡片、时间轴，不要复杂三维图

## 二、内文配图

### 先决定该不该配图

写作完成后，逐个 H2 判断是否需要配图：

| 适合配图 | 不适合配图 |
|----------|------------|
| 有真实数据、对比、排名、时间线 | 纯观点输出且篇幅很短 |
| 有场景还原价值 | 紧接上一张图，间距不足 300 字 |
| 有转折、高点、案例节点 | 结尾 CTA 段落 |
| 单段过长，需要调节阅读节奏 | 已有引用块或代码块，视觉信息足够 |

### 配图位置约束

- 总数 3-6 张
- 1500 字左右配 3 张
- 2000 字左右配 4 张
- 2500 字左右配 5-6 张
- 相邻两张图至少间隔 300 字
- 不要在文章第一段之前放图
- 配图插在对应段落之后，不是之前

### 内文配图输出格式

每张图必须输出以下字段：

```markdown
### 配图 {序号}: 位于「{H2标题}」第 {N} 段后
- 槽位编号：img-0{序号}
- 目标文件名：img-0{序号}.jpg
- Markdown 引用：![配图 {序号}：{alt 文案}](img-0{序号}.jpg)
- alt 文案：{一句自然中文，说明这张图是什么}
- 图例说明：{放在图片正下方的一小行解释，建议格式“图 {序号}：一句话解释这张图在说明什么”}
- 配图目的：{信息强化 / 场景还原 / 节奏调节 / 数据说明}
- 对应内容：{这一段在讲什么，用 1 句话说清}
- 画面核心：{读者第一眼应该看到什么}
- 必须出现的数据：
  - 指标名：{数值}，单位：{单位}，时间：{时间}，来源：{来源}，位置：{左上/中部/右下...}
  - 指标名：{数值}，单位：{单位}，时间：{时间}，来源：{来源}，位置：{左上/中部/右下...}
- 必须出现的中文：
  - 主标题："{中文标题}"
  - 图例标签："{标签1}"、"{标签2}"
  - 说明文字："{如需要}"
- 图表或画面结构：{柱状图/折线图/卡片式数据/场景插画/物品特写/人物动作等}
- 构图：{16:9 横版，主体和数据模块分别在哪}
- 色调与风格：{与封面保持一致}
- 中文排版要求：{文字大小、密度、层级、可读性要求}
- 正向提示词：
  "{一整段超详细中文提示词，必须包含主体、数据、中文文案、图表类型、排版、材质、光线、镜头和风格}"
- 负向提示词：
  "{明确排除乱码、错别字、英文乱入、无意义图表、小字堆叠、错误数据映射、信息拥挤}"
- 免费图库备选关键词：{给 Unsplash / Pexels / 稿定等平台检索用}
- 后期覆字说明：{如果需要，列出建议手工覆盖的中文内容}
```

### 内文配图写法要求

- 尺寸统一为 **16:9 横版**
- 风格和封面必须同一套色调体系
- 优先做“读手机也看得懂”的图
- 数据图优先用简单结构，不要塞满复杂坐标轴和细小注释
- 重要数据最多突出 3-6 个点
- 如果正文是对比关系，图里也要把对比关系做出来，而不是只有装饰元素
- 生成提示词后，必须把对应的 Markdown 图位实际插进正文，而不是只在提示词文件里描述
- 每张图插入正文时，必须在图片下方紧跟一行图例说明，例如：`*图 1：中美模型周调用量对比，突出中国模型近四周持续领先。*`
- 对应占位图必须落到文章目录，保证预览时已经能看到版面中的图位

## 三、必须使用的措辞模板

### 中文正确性模板

在所有需要文字的图片正向提示词里，直接加入这句话：

```text
画面中的所有文字必须使用简体中文，中文要自然准确、完整清晰、排版工整、笔画正常，不能出现乱码、错别字、异体字、繁简混用、拼音代字、无意义符号。
```

### 数据正确性模板

在所有需要图表或数字的图片正向提示词里，直接加入这句话：

```text
图中的数字、年份、百分比、标签名称、来源标注都必须与给定数据一致，不要擅自改写，不要省略关键单位，不要生成与正文不一致的虚构数据。
```

### 负向提示词模板

```text
不要乱码中文，不要错别字，不要伪汉字，不要繁简混用，不要英文乱入，不要小字堆积，不要低清晰度，不要脏背景，不要过度装饰，不要错误图表比例，不要虚构数据，不要错误年份，不要无意义图标。
```
